# GA AI Insurance Compare

## 1. 프로젝트 개요
보험모집인/GA 설계사가 고객 상담 전에 여러 보험사의 상품을 비교하고, 고객 니즈에 맞는 추천 상품·상담 포인트·제안서 요약을 확인하는 AI 기반 웹서비스 PoC 데모입니다.

## 2. 서비스 목적
- 고객 상황 기반 보장 니즈 분석
- 여러 보험상품의 장단점 비교
- 한화생명 상품의 강점을 고객 니즈와 연결해 설명
- 상담 멘트와 제안서 작성 시간 단축

## 3. 주요 기능
- 메인 대시보드 KPI 및 오늘의 추천 고객 TOP 5
- 고객 니즈 점수 분석
- 보험 큰 범위별 상품 비교 및 고객 설명 항목 중심 비교표
- mock scoring 기반 AI 추천 랭킹
- 비교 분석 리포트 및 한화생명 상담 포인트
- 상담 멘트 자동 생성
- 자연어 보험 니즈 keyword 분석
- 제안서 요약 UI

## 4. 화면 구성
사이드바 메뉴는 대시보드, 고객/니즈 분석, 상품 비교, 추천·상담 제안, 서비스 소개로 단순화했습니다. 상품 비교 화면은 가족보장, 건강·3대질병, 노후·연금, 간병·치매, 생활보장처럼 큰 보험 범위별로 진입한 뒤 고객 설명 항목 중심의 표로 비교하도록 구성됩니다.

## 5. mock 데이터 설명
`src/data/customers.ts`에는 10명의 고객 mock 데이터가 있으며 가족구성, 기존 보험, 건강 이슈, 최근 이벤트, 관심사, 보험료 민감도 등을 포함합니다. `src/data/products.ts`에는 22개 상품 mock 데이터가 있으며 한화생명 상품 6개와 타사 상품을 포함합니다. 모든 보험료, 보장점수, 환급률 관련 값은 시연용 예시입니다.

## 6. 추천 로직 설명
`src/utils/recommendationEngine.ts`는 고객의 가족 여부, 건강검진/질병 키워드, 노후/연금 키워드, 간병/치매 키워드, 보험료 민감도와 추천 관점을 조합해 상품 점수를 계산합니다. 외부 AI API는 호출하지 않습니다.

## 7. 한화생명 상품 강조 방식
한화생명 상품은 전략상품 badge와 상담 포인트를 더 명확하게 보여주지만 무조건 1위가 되도록 조작하지 않습니다. “한화생명 우선 검토” 관점에서는 소폭 가중치만 부여하며, 타사 상품이 보험료·암보장·연금 등 특정 기준에서 더 높은 점수를 보일 수 있습니다.

## 8. 실행 방법
```bash
npm install
npm run dev
npm run build
npm run lint
```

## 9. 시연 시나리오
1. 대시보드에서 오늘 추천 고객 확인
2. 고객 니즈 분석 화면에서 고객 선택
3. 고객별 보장 니즈 점수 확인
4. 상품 비교 화면에서 가족보장/건강·3대질병/노후·연금 등 큰 범위 선택
5. 고객이 궁금해하는 월 보험료, 납입/보장 기간, 추천 고객, 강점/주의점, 상담 포인트를 표로 확인
6. AI 추천 랭킹에서 고객에게 적합한 상품 TOP 5 확인
7. 한화생명 상품의 추천 근거와 타사 대비 차별점 확인
8. 상담 멘트 생성
9. 제안서 요약 화면 확인
10. 서비스 소개 화면에서 실제 운영 확장 방향 설명

## 10. 실제 운영 확장 방향
- 실제 상품 DB 및 약관 DB 연동
- 고객/계약 데이터 및 상담 이력 연동
- GA 모집인별 추천 상품 관리
- 영업관리자 대시보드
- 생성형 AI 기반 제안서 자동 작성
- 상품 개정 시 비교 기준 자동 업데이트

> 본 프로젝트의 데이터는 모두 시연용 예시 데이터이며 실제 판매자료가 아닙니다.

## 11. 클라우드 배포 방법

### GitHub Pages 자동 배포
이 저장소에는 `.github/workflows/deploy-github-pages.yml` 워크플로가 포함되어 있습니다. `main` 또는 `master` 브랜치에 push하면 GitHub Actions가 자동으로 다음 과정을 실행합니다.

1. `npm ci`로 의존성 설치
2. `npm run build`로 `dist/` 생성
3. `dist/`를 GitHub Pages artifact로 업로드
4. GitHub Pages에 배포

GitHub 저장소에서 최초 1회만 **Settings → Pages → Build and deployment → Source: GitHub Actions**로 설정하면 됩니다.

### Vercel 자동 배포
이 저장소에는 `vercel.json`이 포함되어 있어 Vercel에서 프로젝트를 import하면 자동으로 다음 설정을 사용합니다.

- Framework: Vite
- Install Command: `npm ci`
- Build Command: `npm run build`
- Output Directory: `dist`

Vercel에서는 GitHub 저장소를 연결한 뒤 기본값 그대로 Deploy를 누르면 됩니다.

### 로컬에서 배포 결과 미리보기
```bash
npm run build
npm run preview
```
